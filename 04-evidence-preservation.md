# Evidence Preservation Policy

## I. Purpose

This policy establishes the records that should be preserved to support attorney review, board review, public-disclosure consistency, and court-ready documentation.

The goal is to preserve contemporaneous evidence of intent, design boundaries, public statements, legal-review checkpoints, and risk controls.

## II. Records to Preserve

GFO should preserve:

- Board resolutions
- Adoption records
- Versions of website copy
- Versions of book descriptions
- Token disclaimers
- Counsel-review memos
- Smart-contract audit records
- Treasury wallet policies
- Non-custody policies
- Community moderation rules
- Screenshots of public disclosures
- Version-controlled GitHub records
- DAO proposal records
- Risk committee notes
- Tax-review records
- Communications policy records
- Incident-response records
- Wallet-access logs for project wallets
- Conflicts-of-interest disclosures
- Founder compensation disclosures, if any
- Public correction logs

## III. GitHub Preservation Rules

No legal-policy file should be deleted without recording the reason in `CHANGELOG.md`.

No legal-policy history should be rewritten using force-pushes after public release.

Each material change should be committed separately with a clear commit message.

Each file should include a version history through `CHANGELOG.md`.

## IV. Public Disclosure Snapshots

Before any public token-related announcement, GFO should preserve:

- Screenshot of website disclaimer
- Screenshot of book description
- Screenshot of social post
- Screenshot of GitHub README
- Screenshot of non-custody policy
- Date and time of publication
- Person responsible for publication
- Version number of relevant disclosure

## V. Legal Review Records

If counsel reviews a file, preserve:

- Date of review
- Reviewer name or firm
- Reviewed version number
- Comments received
- Changes made
- Approval status
- Any unresolved risks

## VI. Incident Preservation

If a public misunderstanding occurs, preserve:

- Original statement
- Correction issued
- Date correction was issued
- Platforms corrected
- Internal notes explaining correction
- Updated policy language

## VII. Final Rule

The goal is contemporaneous documentation, not after-the-fact cleanup.

**GFO coordinates the experiment. GFO does not custody the public.**
