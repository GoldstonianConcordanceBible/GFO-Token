# Token Risk Controls

## I. Purpose

This document establishes securities, custody, money-transmission, tax, public-communications, and operational risk controls for any GFO token-related activity.

The purpose is not to declare that any structure is legally compliant.

The purpose is to document a compliance-aware posture and preserve a record that GFO is designed to avoid custody, investment promises, pooled public assets, managed accounts, redemption guarantees, and public asset-management activity.

## II. Securities-Law Risk Controls

GFO communications and design should avoid facts that may support an investment-contract analysis.

GFO must not market any token as:

- An investment
- A security
- A share
- An equity interest
- A debt instrument
- A revenue-share right
- A dividend right
- A profit right
- A treasury claim
- A family-office ownership claim
- A managed financial product
- A pooled investment vehicle

GFO must avoid language suggesting that token holders may expect profits from the efforts of Dr. Justin Goldston, the Goldston Family Office, developers, affiliated entities, the DAO, treasury managers, community managers, or any identifiable core team.

## III. Prohibited Economic Claims

No GFO representative may state or imply:

- Token holders will profit
- Token holders will receive yield
- Token holders will receive dividends
- Token holders will receive revenue
- Token holders will receive buybacks
- Token holders will receive guaranteed grants
- Token holders will receive guaranteed redemption
- Token holders own the family office
- Token value is backed by family-office assets
- Token value will increase because of founder, team, or treasury action
- Liquidity will be guaranteed
- Listings will be guaranteed
- A price floor will be maintained

## IV. Money Transmission and MSB Controls

GFO shall not accept and transmit value on behalf of others unless counsel determines that the activity is lawful, registered, exempt, or otherwise approved in writing.

GFO shall not operate:

- Hosted wallets
- Public custody accounts
- Payment routing for users
- Redemption pools
- Fiat off-ramps
- Exchange services
- Transfer services for the public
- Managed bridge services
- Public asset pooling

Any proposed feature that could involve accepting, transmitting, converting, redeeming, or routing value for others must be paused pending legal review.

## V. Tax Boundary

GFO materials must not describe any token as legal tender, cash, deposit value, guaranteed redemption value, or currency substitute.

Preferred language:

- Digital asset
- Token
- Participation artifact
- Access layer
- Cultural access
- Educational access
- Governance signal
- Research object

Avoid:

- Currency
- Cash
- Deposit
- Money substitute
- Guaranteed book credit
- Guaranteed redemption value

Participants are responsible for their own tax reporting, accounting records, cost basis, gains, losses, transfers, and jurisdiction-specific obligations.

## VI. Operational Risk Controls

Before any token-related feature is activated, GFO should document:

- Feature description
- Custody analysis
- Securities-risk analysis
- Money-transmission analysis
- Tax-risk note
- Public-disclosure language
- Smart-contract risk note
- Third-party dependency note
- Approval record
- Version number
- Date of activation
