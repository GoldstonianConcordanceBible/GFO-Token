Legal Review Protocol

I. Purpose

This protocol governs review, approval, and revision procedures for the Goldston Family Office Token Legal Framework.

The purpose is to preserve a court-ready record showing that legal-risk controls, non-custody boundaries, public-communications standards, and token-risk controls were reviewed before public deployment, publication, or material change.

III. Review Classes

Class A — Board-Record Changes

Board-record changes include edits to:

* Governance resolutions
* Adoption records
* Custody boundaries
* Token characterization
* Public disclaimer language
* Legal status statements
* Escalation procedures

Class A changes require board or authorized representative review before publication.

Class B — Counsel-Review Changes

Counsel-review changes include any language involving:

* Securities law
* Investment adviser status
* Custody
* Money transmission
* Tax treatment
* DAO governance
* Token redemption
* Treasury-linked benefits
* Exchange or bridge activity
* Grants funded by token activity
* Public asset pooling
* User funds
* User wallets

Class B changes should be reviewed by qualified legal counsel before public use.

Class C — Editorial Changes

Editorial changes include spelling, formatting, internal cross-references, folder paths, and non-substantive copy edits.

Class C changes may be made without counsel review if they do not alter legal meaning.

IV. Required Review Questions

Before approving any material change, answer:

1. Does this change imply GFO provides custody?
2. Does this change imply GFO manages public assets?
3. Does this change imply token holders may profit from GFO’s efforts?
4. Does this change imply redemption, yield, liquidity, buybacks, or revenue sharing?
5. Does this change imply GFO is a public family office?
6. Does this change imply GFO is an adviser, broker, exchange, custodian, bank, trust company, or money transmitter?
7. Does this change imply legal compliance has already been determined?
8. Does this change require counsel review before publication?

V. Required Record

Each material change should record:

* Date
* File changed
* Version number
* Change summary
* Reviewer
* Approval status
* Whether counsel review was required
* Whether board review was required
* Link to commit hash

VI. Final Rule

GFO coordinates the experiment. GFO does not custody the public.
